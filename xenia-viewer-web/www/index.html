xenia-wire — browser demo

Pre-alpha PQC-sealed wire protocol, running in WebAssembly.

PRE-ALPHA — DO NOT USE IN PRODUCTION. This page is a reference demo that the Xenia wire works in a browser. It seals and opens a frame locally in WASM — there is no network, no server, no real session. See the repo for the crate, spec, and paper.

Demo


What's happening

  1. JS calls new WasmSession() twice (sender + receiver).
  2. Both sides install the same 32-byte fixture key via installKey.
  3. Sender calls sealFrame with the payload above — ChaCha20-Poly1305 encrypts into an envelope.
  4. Receiver calls openFrame on the envelope bytes — AEAD verifies, replay window advances, Frame deserializes.
  5. A second openFrame on the same bytes is rejected by the replay window.

The entire roundtrip runs in WASM; open DevTools and look at console.log for per-step diagnostics.
