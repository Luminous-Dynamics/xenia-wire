Xenia

A PQC-ready, decentralized-trust wire protocol for remote control. Open specification. Auditable reference implementation. Pre-alpha.

PRE-ALPHA — DO NOT USE IN PRODUCTION. The wire format is not yet frozen. Breaking changes will land between 0.1.x releases. No formal cryptographic review has happened. The handshake layer is a placeholder. This is a research artifact seeking review, not a product.

Why

Modern MSP remote-control tools — ConnectWise, TeamViewer, AnyDesk — concentrate identity, authority, and authentication in a single vendor. When any of the three is compromised at the vendor layer, every downstream MSP is compromised in parallel. CVE-2024-1709 is the pattern's canonical demonstration: an authentication-bypass in ScreenConnect was weaponized within 24 hours and used to deploy ransomware across dozens of MSP client networks through a single vendor hole. Patching the CVE doesn't alter the topology. The topology is the problem.

Xenia specifies a wire that cleanly separates symmetric crypto on the byte level from trust topology. The wire is open-spec and byte-deterministic across implementations. The trust topology is a separate, composable layer. You can run Xenia with a centralized identity provider today; you can swap it for a decentralized one tomorrow, with no wire-format change.

Quick install

$ cargo add 'xenia-wire@0.2.0-alpha.3'

Then a minimal seal/open roundtrip: 

use xenia_wire::{Session, seal_frame, open_frame, Frame};

let key = [0xAB; 32];
let mut sender = Session::new();
let mut receiver = Session::new();
sender.install_key(key);
receiver.install_key(key);

let frame = Frame {
    frame_id: 1,
    timestamp_ms: 1_700_000_000_000,
    payload: b"hello, xenia".to_vec(),
};
let sealed = seal_frame(&frame, &mut sender).unwrap();
let opened = open_frame(&sealed, &mut receiver).unwrap();
assert_eq!(opened.payload, b"hello, xenia");

Seeking review

The protocol is pre-alpha. I'm specifically looking for:

  • Cryptographers — a 15-minute check on SPEC §3 (nonce layout) and §5 (replay window). The goal is to catch composition issues before the wire format freezes.
  • MSP operators — a reality check on the threat model in §6.2 of the paper. Does the decentralized-trust posture match how you actually think about risk?
  • Implementers in other languages — use the byte-deterministic test vectors to validate a Go / Swift / Python / Zig client. Divergence is useful information either way.

Open an issue for design feedback. For security-sensitive reports, please use the private advisory channel instead.

Xenia (ξενία) — the ancient Greek covenant between guest and host. A technician is a guest in a client's machine; the client extends bounded hospitality; the protocol codifies the terms cryptographically.
