Vector 07: consent_request (draft-03, feature = consent)
--------------------------------------------------------
A ConsentRequest for ScreenAndInput scope, signed with a
deterministic Ed25519 seed. An alternate-language
implementation can reproduce every byte from the fixture
seed + the fixed session parameters.

Fixture parameters:
source_id  = "XENIATST"
epoch      = 0x42
key        = fixture-key (shared with vectors 01-05)
payload_type = 0x20 (CONSENT_REQUEST)

Ed25519 signing seed: "xenia-consent-test-vector-seed!!" (32 bytes)

ConsentRequest fields (draft-03):
request_id          = 7
requester_pubkey    = derived Ed25519 public key
session_fingerprint = 5b94fb75dd4d499825c7f26f32dea7dce067d59a5200584a2c9d7d9e18dfd7d4
valid_until         = 1700000300
scope               = ScreenAndInput (= 1)
reason              = "xenia test vector"
causal_binding      = None (draft-03 MUST be None)

session_fingerprint is HKDF-SHA-256(session_key, info =
source_id || epoch || request_id_be). Both peers derive it
locally from their own copy of the session key; receivers
reject requests whose embedded fingerprint does not match.
See SPEC §12.3.1 for the derivation.

.input.hex is the bincode-encoded ConsentRequest (including
the 64-byte signature). .envelope.hex is the AEAD-sealed
form sealed under the fixture key.

To validate: decrypt envelope with fixture key + extracted
nonce; bincode-deserialize; derive session_fingerprint
locally from the fixture key and compare; verify Ed25519
signature over bincode(core).
